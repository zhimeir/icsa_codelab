Selective Inference Method Examples

webR code cells for the ICSA selective inference talk

This file uses Quarto Live webR cells. Install the extension once in this directory with quarto add r-wasm/quarto-live, then render with quarto render icsa_selinf_webr_examples.qmd.

All examples use base R so they can run in the browser without extra webR package downloads.

Setup

This setup cell defines the simulated p-value generator and the three global null testing functions used later. Run it first so the later examples can reuse the same data-generating model and method implementations.

Global Null Tests

The first lecture compares ways to test a global null, H_global = H1 intersect H2 intersect ... intersect Hm.

This chunk generates one dependent collection of p-values with a few embedded signals, then applies Bonferroni, Fisher, and Cauchy to the same input. Compare the combined p-values to see how each method responds to sparse evidence.

This plot makes the Bonferroni rule visible. Green bars are signal hypotheses, blue bars are null hypotheses, and the dashed red line is the cutoff alpha / m.

This simulation repeats the global-null comparison many times. The null runs estimate type-I error, while the runs with signals estimate power for the same three methods.

This chunk is a small power-curve sandbox. Change sparsity_grid, rho_grid, or signal_strength to test other regimes; the plot shows the rejection rate as the signal strength increases for each sparsity-correlation setting.

FWER Procedures

The FWER slides include Bonferroni, Sidak, and Holm. Bonferroni and Sidak are single-step rules; Holm is a step-down rule that starts at the smallest p-value.

This helper cell defines the FWER procedures. The holm_steps() table is especially useful for teaching because it shows the ordered p-values and the changing Holm threshold at each step.

This example applies the three FWER rules to the same eight p-values. Holm can reject more hypotheses than Bonferroni while still controlling the family-wise error rate.

This Monte Carlo cell estimates how often each rule makes at least one false rejection. The unadjusted rule is included as a baseline to show why multiplicity correction is needed.

Lecture 3: FDR, BH, and Knockoffs

Lecture 3 moves from avoiding any false discovery to allowing a controlled fraction of false discoveries. The code below follows the lecture sequence: FDP/FDR, the BH step-up rule, dependence and adaptive variants, ordinary feature-selection p-values, and finally knockoffs.

This first chunk separates the random false discovery proportion from its expectation. For one study we can compute V, R, and FDP; FDR is the average of this quantity over repeated studies.

This helper cell implements the BH procedure, the Benjamini-Yekutieli correction for arbitrary dependence, and a simple Storey-BH variant using the lecture’s p_j > 1/2 null-proportion estimate.

This chunk shows BH exactly as a step-up procedure: sort the p-values, draw the line q * j / m, and reject through the largest rank that falls below the line.

This simulation mirrors the lecture’s guarantee discussion. BH is compared with the more conservative BY correction and the adaptive Storey-BH rule as correlation among z-statistics increases.

This chunk focuses on the lecture’s null-proportion idea. Storey-BH estimates the fraction of nulls from the right side of the p-value histogram and uses that estimate to make BH less conservative.

The lecture frames controlled feature selection as a multiple-testing problem. Here, each coefficient in a linear model gets a t-test p-value, then BH selects features at the target FDR level.

This helper cell contains a compact fixed-X knockoff construction for small examples, the knockoff threshold, and a simple importance statistic W_j based on whether the original feature is more associated with Y than its knockoff.

This example follows the knockoff slides end-to-end: construct knockoffs, form antisymmetric feature-importance statistics, choose the threshold, and report the selected features.

Simultaneous Coverage

The simultaneous coverage slides ask for intervals that cover all targets at once, not just one target at a time.

This chunk compares ordinary marginal intervals with Bonferroni and Sidak simultaneous intervals. The plot focuses on Bonferroni intervals and marks the true target values so coverage can be checked visually.

This simulation estimates family coverage: the probability that every interval in the family covers its target at the same time. Marginal intervals usually miss this simultaneous target when there are many outcomes.

Post-selection Inference

This toy example contrasts a naive interval after choosing the largest effect with a simple sample-splitting interval. Sample splitting is not the only selective inference method, but it makes the selection-estimation separation visible in a short browser-friendly example.

This final chunk shows the selection problem in a minimal setting. The naive interval uses the same data to choose and estimate a variable, while the sample-split interval chooses on one half and estimates on the other half.
